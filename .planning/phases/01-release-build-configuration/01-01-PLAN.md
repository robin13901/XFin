  <action>
    Three changes in this task:

    1. **Update .gitignore** - Add these entries at the end under a new "# Release signing" comment block:

       android/key.properties
       android/upload-keystore.jks

    2. **Update pubspec.yaml** - Change `version: 1.0.0+1` to `version: 1.3.0+1`. The versionName 1.3.0 matches the milestone, and versionCode 1 is correct for the first Play Store upload.

    3. **Update android/app/build.gradle.kts** - Replace the entire file content with a version that:
       - Reads `key.properties` from the project root android directory using `val keystoreProperties = Properties()` and loading from `rootProject.file("key.properties")`
       - Wraps the properties loading in a conditional check: only load if the file exists (so debug builds still work without key.properties)
       - Adds a `signingConfigs` block with a `release` config that reads `storeFile`, `storePassword`, `keyAlias`, and `keyPassword` from the loaded properties
       - Updates the `buildTypes.release` block to use `signingConfigs.getByName("release")` instead of `signingConfigs.getByName("debug")`
       - Removes the TODO comments about signing
       - Keeps ALL other existing configuration (namespace, compileSdk, compileOptions, kotlinOptions, defaultConfig, flutter block) exactly as-is
       - Adds `import java.util.Properties` at the top of the file (after plugins block, before android block)

       The key.properties loading pattern (Kotlin DSL):
       ```kotlin
       val keystorePropertiesFile = rootProject.file("key.properties")
       val keystoreProperties = Properties()
       if (keystorePropertiesFile.exists()) {
           keystoreProperties.load(keystorePropertiesFile.inputStream())
       }
       ```

       The signingConfigs block goes inside `android {}`, before `buildTypes`:
       ```kotlin
       signingConfigs {
           create("release") {
               keyAlias = keystoreProperties["keyAlias"] as String? ?: ""
               keyPassword = keystoreProperties["keyPassword"] as String? ?: ""
               storeFile = file(keystoreProperties["storeFile"] as String? ?: "upload-keystore.jks")
               storePassword = keystoreProperties["storePassword"] as String? ?: ""
           }
       }
       ```

       IMPORTANT: Use the null-safe pattern with `as String? ?: ""` defaults so the build doesn't crash when key.properties is missing (debug builds).
  </action>

  <decision>Provide a password for the upload keystore. This password protects the .jks file used to sign your app for Google Play. You will need to remember this password -- store it in a password manager.</decision>
  <context>
    The upload keystore is used to sign the AAB you upload to Google Play Console. Google Play App Signing then re-signs with the final distribution key. If you lose this upload key, you can request a reset from Google, but it takes time. The password should be strong but memorable (or stored in a password manager).

    You need to provide TWO passwords:
    1. **Store password** - protects the keystore file itself
    2. **Key password** - protects the individual key entry

    For simplicity, most developers use the same password for both. You can provide one password to use for both, or two different passwords.
  </context>
  <options>

      <name>Same password for store and key</name>

    </option>
    <option id="different-passwords">
      <name>Different passwords for store and key</name>
      <pros>Defense in depth</pros>

<task type="auto">
  <name>Task 3: Generate upload keystore, create key.properties, verify release build</name>
  <files>android/upload-keystore.jks, android/key.properties</files>
  <action>
    Three steps using the password(s) from Task 2:

    1. **Generate upload keystore** using keytool (ships with Java/JDK):
       ```bash
       keytool -genkey -v \
         -keystore android/upload-keystore.jks \
         -keyalg RSA \
         -keysize 2048 \
         -validity 10000 \
         -alias upload \
         -storetype JKS \
         -storepass {USER_STORE_PASSWORD} \
         -keypass {USER_KEY_PASSWORD} \
         -dname "CN=XFin, OU=Personal, O=RP, L=Unknown, ST=Unknown, C=DE"
       ```
       If `keytool` is not on PATH, check common locations:
       - `$JAVA_HOME/bin/keytool`
       - `$ANDROID_HOME/../jbr/bin/keytool` (Android Studio bundled JDK)
       - The Flutter SDK often bundles Java; check `flutter doctor -v` output for Java path

    2. **Create android/key.properties** with this exact content:
       ```properties
       storePassword={USER_STORE_PASSWORD}
       keyPassword={USER_KEY_PASSWORD}
       keyAlias=upload
       storeFile=../android/upload-keystore.jks
       ```
       Note: `storeFile` path is relative to the `android/app/` directory, so `../android/` goes up to project root then back into android/. Alternatively use an absolute path or `../../android/upload-keystore.jks` -- but the standard Flutter convention is to place the path relative to where build.gradle.kts resolves `file()`, which is the app module directory (`android/app/`). So the correct relative path is `../upload-keystore.jks` (goes from `android/app/` up to `android/` where the keystore lives).

       Correction: Since upload-keystore.jks is in `android/` and `file()` in build.gradle.kts resolves relative to `android/app/`, use `storeFile=../upload-keystore.jks`.

    3. **Verify the keystore was gitignored** by running:
       ```bash
       git status android/upload-keystore.jks android/key.properties
       ```
       Both files should NOT appear as untracked (they should be ignored).

    4. **Build the release AAB** to verify the entire signing pipeline works:
       ```bash
       flutter build appbundle --release
       ```
       Expected output: "Built build/app/outputs/bundle/release/app-release.aab"

       If the build fails with signing errors, debug by:
       - Checking key.properties paths are correct
       - Verifying keytool generated the keystore successfully: `keytool -list -keystore android/upload-keystore.jks`
       - Ensuring build.gradle.kts loads properties correctly

    5. **Verify the AAB is signed** (optional but good confirmation):
       ```bash
       jarsigner -verify build/app/outputs/bundle/release/app-release.aab
       ```
       Should say "jar verified" (or use `apksigner` if available).
  </action>
